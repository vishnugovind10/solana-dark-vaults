# v0.1.0

`v0.1.0` is the first evidence-bearing reference release of `solana-dark-vaults`. It separates deterministic confidential-allocation source, public settlement accounting, and an off-chain coordination simulator while keeping every unimplemented protocol boundary explicit.

## Implemented

- Anchor vault state with checked aggregate share math, pause control, pending epochs, commitment validation, callback-authority checks, and exact 10,000-bps settlement invariants.
- A fixed-three-pool Arcis encrypted allocation definition and a separately testable cleartext reference implementation with 24 generated fixtures.
- A TypeScript coordinator with deterministic oracle input, local X25519/AES-256-GCM transport, crash-safe epoch state, idempotent settlement, and a metered local x402 challenge model.
- Pinned portable and hosted-Linux workflows, guarded local-cluster and devnet scripts, an executable documentation verifier, and an adversarial self-audit.

## Mocked or not yet integrated

- Pool positions are deterministic in-memory mocks; no Kamino, JupLend, Maple, or token-program CPI is present.
- The portable encrypted transport is not an Arcium MXE execution.
- x402 payments use an in-process facilitator and move no funds.
- Depositor custody, Token-2022 confidential transfers, computation-definition accounts, and Arcium-verified callback accounts are not implemented.
- The devnet script has not been executed and no devnet computation is claimed.

## Verification summary

- 14 portable tests: 7 Anchor/state host tests, 4 allocation reference tests covering 24 generated fixtures, and 3 coordinator tests.
- Coordinator line coverage: 95.63%; function coverage: 100%.
- `npm audit --audit-level=moderate`: zero vulnerabilities after the scoped `jayson > uuid@11.1.1` lockfile override.
- Portable CI and lint: [CI run](https://github.com/vishnugovind10/solana-dark-vaults/actions/runs/29596505916) and [lint run](https://github.com/vishnugovind10/solana-dark-vaults/actions/runs/29596506594) passed on commit `ecaca16`.
- Arcium CLI build/local-cluster evidence: [Arcium run](https://github.com/vishnugovind10/solana-dark-vaults/actions/runs/29596505790) passed on commit `ecaca16`, including devcontainer verification, `arcium build --skip-keys-sync`, and `arcium test`.
- Arcium evidence boundary: the run proves the configured hosted Ubuntu build and two-node local-cluster reference tests. It does not prove a devnet deployment, production recovery set, or end-to-end confidential callback round.
- Self-audit: `docs/audit/2026-07-self-audit.md`; open findings are tracked with the `hardening` label.

## Known limitations

- Arcium has no native Windows CLI; the real toolchain path requires Linux, WSL2, or hosted Ubuntu.
- Anchor is pinned to the official Arcium 0.13.x prerequisite, version 1.0.2, because Anchor 1.1.2 and Arcis 0.13.2 resolve incompatible `digest` 0.11 dependency lines in the CLI-discovered workspace.
- Callback settlement is signer-authorized but not yet bound to Arcium computation proof/accounts.
- Aggregate share accounting has no per-depositor custody state and must not be connected to real assets.
- Settlement weights and timing are public after callback execution.
- The two-node local Cerberus topology is a functional test configuration, not an operational decentralization claim.

See `docs/limitations.md` and `docs/threat-model.md` for the complete assurance boundary.
