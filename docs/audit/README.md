# Audit policy

This repository has not been audited by a third party. It is a reference architecture, not an assurance opinion or a substitute for an independent review.

Adversarial self-audits are run against material changes and published in this directory. A self-audit records evidence and unresolved design risk; it does not certify the program. Open findings are tracked in GitHub issues with the `hardening` label. A finding is closed only after the fix and its regression test are merged.

Reports are dated snapshots. Reviewers should verify line references against the commit named in each report.
